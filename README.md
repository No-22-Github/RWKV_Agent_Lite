# RWKV_Agent_Lite

RWKV_Agent_Lite 是一个面向 **RWKV 模型 Agent 能力验证** 的轻量项目。仓库名是
`RWKV_Agent_Lite`，但当前实际启动入口是 **`rvoone`**。

它来自我的个人作品 **EvoOne**（*One system, growing with you*）。EvoOne 的目标并不是一个
简单的对话程序，而是一个更高阶的个人 Agent 助理：更自动、更自主，也更强调持续协作和长期成长。
不过 EvoOne 目前还没有做完，所以还没有正式开源。

这个仓库可以理解为从 EvoOne 中抽离出来的一个 **Lite 验证版本**。我把其中最核心的一部分单独拿出来，
保留 Agent 循环、工具调用和关键执行链路，去掉完整产品里暂时不影响验证结论的部分，目的是用一个更小、
更清晰、也更容易反复实验的项目，去观察 **RWKV 模型是否具备工具调用能力，以及在 Agent loop 中持续工作的能力**。

相较于完整系统，这个 Lite 版本更关注几个很具体的问题：

- 在尽量少的上下文中，模型能不能正确理解任务并进入 Agent 循环。
- 在工具数量被严格控制的前提下，模型能不能稳定完成调用、读取结果和继续决策。
- 当系统被精简到足够轻量时，是否能进一步降低 token 消耗，提升实验效率和可复现性。

因此，这个项目不是在追求“大而全”，而是在追求 **核心能力足够纯粹、验证路径足够清楚**。目前项目只保留
**2 个工具**，分别是：

- `exec`：用于执行命令，验证模型在工具调用后的操作与反馈处理能力。
- `web_fetch`：用于抓取网页内容，验证模型对外部信息获取与继续推理的能力。

在这样的最小工具集下，首轮 token 消耗通常可控制在 **600 token 以内**，更适合做能力适配、行为观察和迭代实验。

## Why this project exists

这个项目不是 EvoOne 的开源替代品，而是一份更偏向工程实验的抽离版本。它存在的意义主要有三点：

- 验证 RWKV 在真实 Agent 流程中的可用性，而不是只看单轮对话表现。
- 观察轻量工具调用链路是否稳定，尤其是“理解任务 → 调用工具 → 读取结果 → 继续执行”这一闭环。
- 为 EvoOne 后续能力演进提供一个更小、更容易调试、更容易反复试验的实验基座。

## Quick start

要求：**Python 3.11+**。

```bash
python -m pip install -e ".[dev]"
```

安装后可以通过以下方式启动：

```bash
rvoone --help
```

或者：

```bash
python -m rvoone
```

如果需要初始化本地配置与工作目录：

```bash
rvoone onboard
```
